RecycleView - part1

发表于 2015-07-25   |  

  RecycleView其实已经出来很久了，而且被越来越广泛的用来替代ListView。RecycleView就像名字，是可回收的。在使用listview的时候，会去复用convertView和viewholder来实现优化效率。在RecycleView中是强制使用viewholder。它只关心回收和复用的工作而不去关心视图相关的工作。在使用listview中，视图，数据，回收等操作显得过于耦合，这就是为什么要去设计这样一个工具。

  在RecycleView中，是使用像”插件化”的方式剥离这些复杂的耦合。布局方面，只要去关心LayoutManager，想要item动画，关心ItemAnimator就可以了，这就使使用十分的灵活。

  这篇先概括的介绍一下如何去使用RecycleView。

在XML文件中定义

这个和使用一般的控件一样，没什么特别的，不贴代码了。

获取配置RecycleView

这个也没什么好说的，就是像一般的findviewbyid()。这里还可以去设置recycleview的一些配置。

  mRecycleView.setLayoutManager(new LinearLayoutManager(this, LinearLayoutManager.VERTICAL, false));
mRecycleView.setItemAnimator(new DefaultItemAnimator());
mRecycleView.addItemDecoration(new DividerItemDecoration(this,DividerItemDecoration.VERTICAL_LIST));

LayoutManager是设置布局方式，目前有StaggeredGridLayoutManager瀑布流和默认的LinearLayoutManager。都可以设置水平或者垂直。当然也可以自定义自己的LayoutManager，这个以后再去介绍。setItemAnimator这个是用来设置item添加或者删除时候的动画，默认给了一个DefaultItemAnimator。addItemDecoration是给每个item添加偏移量、可以给item间添加间距或者高亮条目，添加其他修饰效果等等，这个可以添加很多个效果同时生效。

创建Adapter

RecycleView是继承自ViewGroup，不能再像使用listview时候使用baseadapter等adapter了，这里需要继承自RecyclerView.Adapter，还有传入一个ViewHolder，这个ViewHolder也要继承自RecyclerView.ViewHolder

class MyViewHolder extends RecyclerView.ViewHolder {
       TextView tv;

       public MyViewHolder(View itemView) {
           super(itemView);
           tv = (TextView) itemView.findViewById(R.id.id_num);
       }
   }

像这样去创建adapter

class MyRectcleAdapter extends RecyclerView.Adapter<MyRectcleAdapter.MyViewHolder>

还要去实现几个必须的方法：

  • 构造方法，这个不用说
  • getItemCount 这个返回也不用说了 和listview中一样
  • onBindViewHolder 这个是绑定viewholder,就是在这里去处理一些view的赋值等等
  • onCreateViewHolder 这个是创建viewholder这里将item的布局文件传进去

这样基本就完成了adapter，和listview有点差别，但是可控制的地方更多，当然要自己写的代码也多了。具体代码就不贴了，太长也没意思，给个地址 GitHub
