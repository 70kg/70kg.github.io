Android属性动画

发表于 2015-07-03   |   分类于 动画   |  

  其实之前就学习过关于android动画的一些东西,但是只是为了解决当时的问题简单的学习了一下，没有系统的学习，时间久了就忘记了，这段时间再从新系统的学习一下动画。

属性动画

  关于Android动画的部分，我决定直接学习属性动画，之前的帧动画等用处没有属性动画那么的广泛和强大。好吧，现在开始聊聊属性动画。这篇还是讲基础。

ObjectAnimator

  ObjectAnimator是属性动画的执行类，使用起来也是很简单，比如让一个view围绕y轴旋转

ObjectAnimator.ofFloat(mTestImage,"rotationY",0.0f,360f).setDuration(500).start();

顺便提一下，属性动画改变的对象真正的属性，像View的Animation动画只是改变了”表面”的属性，比如使用Animation移动一个buton到一个新的位置，但是点击事件的响应还是在原来的位置，这样就十分的不方便。属性动画则不会有这样的问题。
上面的代码只是改变了一个属性，如果想改变两个属性怎么办？可以使用AnimatorSet。这个后面会说到。我们也可以使用ObjectAnimator来实现改变多个属性。

ObjectAnimator animator = ObjectAnimator.ofFloat(mTestImage, "abc", 1f, 0f)
           .setDuration(500);
   animator.start();
   animator.addUpdateListener(new ValueAnimator.AnimatorUpdateListener() {
       @Override
       public void onAnimationUpdate(ValueAnimator valueAnimator) {
           float cVal = (Float) valueAnimator.getAnimatedValue();
           mTestImage.setAlpha(cVal);
           mTestImage.setScaleX(cVal);
           mTestImage.setScaleY(cVal);
       }
   });

我们可以看出，上面的代码中是随便写了一个属性的名称，我们只是取用了valueAnimator的值，这个值是个0-1的值，在设置的持续时间中变化。然后根据valueAnimator的值动态的去改变对象的属性，这样就可以使用ObjectAnimator来同时改变多个属性了。不过还是使用AnimatorSet比较方便。

这里更新一下关于ObjectAnimator的说明，要真正使ObjectAnimator对对象起作用，是要求这个对象有getxx()和setxxx()方法的。get方法其实是可选的，在

ObjectAnimator.ofFloat(mTestImage, "abc", 1f, 0f)

这段代码中，设置了开始值为1f,如果不设置，就会去调用get方法去获取当前的状态。上面给的属性是”abc”，很明显imageview中没有这个属性，其实如果不设置属性的话，ObjectAnimator是可以执行的，不过没有任何的效果，这就是上面还要在onAnimationUpdate（）回调中手动去更新的原因。这里的使用方式和ValueAnimator相似，可以去下面查看一下ValueAnimator的使用。

PropertyValuesHolder

这个看名称就知道，这是用于储存数值的”容器”，使用方法如下：

PropertyValuesHolder p1 = PropertyValuesHolder.ofFloat("alpha", 1.0f, 0f, 1.0f);
   PropertyValuesHolder p2 = PropertyValuesHolder.ofFloat("scaleX", 1.0f, 0f, 1.0f);
   PropertyValuesHolder p3 = PropertyValuesHolder.ofFloat("scaleY", 1.0f, 0f, 1.0f);
   ObjectAnimator.ofPropertyValuesHolder(mTestImage, p1, p2, p3).setDuration(500).start();

ValueAnimator

这个也是动画的执行类，用法和ObjectAnimator差不多，但是更加的灵活。
先看一下代码

 ValueAnimator animator = ValueAnimator.ofFloat(0, mScreenHeight- mBlueBall.getHeight());  
animator.setTarget(mBlueBall);  
animator.setDuration(1000).start();

这段代码是让一个小球垂直的移动，但是我们发现，这里并没有指定要操作的属性，这就是和ObjectAnimator的差别。这样的做法可以让我们去操作那些没有get和set方法的属性，我们知道，使用ObjectAnimator操作的属性是都有get和set方法的。比如我们写一个让小球抛物线运动的代码：

ValueAnimator valueAnimator = new ValueAnimator();
  valueAnimator.setDuration(1000);
  valueAnimator.setObjectValues(new PointF(0, 0));
  valueAnimator.setInterpolator(new LinearInterpolator());
  valueAnimator.setEvaluator(new TypeEvaluator() {
      @Override
      public Object evaluate(float v, Object o, Object t1) {
          PointF p = new PointF();
          LogUtils.e(v);
          p.x = 200 * v * 3;
          p.y = 100 * v * 3 * v * 3;
          return p;
      }
  });
  valueAnimator.start();
  valueAnimator.addUpdateListener(new ValueAnimator.AnimatorUpdateListener() {
      @Override
      public void onAnimationUpdate(ValueAnimator valueAnimator) {
          PointF p = (PointF) valueAnimator.getAnimatedValue();
          mTestImage.setX(p.x);
          mTestImage.setY(p.y);
      }
  });

这里定义了一个TypeValue，根据当前的时间返回一个PointF,然后再UpdateListener去获得当前的值，这个和前面使用ObjectAnimator操作多个属性的方法类似。

动画监听器

为了监听动画执行，可以给动画设置监听器：

ObjectAnimator anim = ObjectAnimator.ofFloat(mBlueBall, "alpha", 0.5f);  

    anim.addListener(new AnimatorListener()  
    {  

        @Override  
        public void onAnimationStart(Animator animation)  
        {  
            Log.e(TAG, "onAnimationStart");  
        }  

        @Override  
        public void onAnimationRepeat(Animator animation)  
        {  
            // TODO Auto-generated method stub  
            Log.e(TAG, "onAnimationRepeat");  
        }  

        @Override  
        public void onAnimationEnd(Animator animation)  
        {  
            Log.e(TAG, "onAnimationEnd");  

        }  

        @Override  
        public void onAnimationCancel(Animator animation)  
        {  
          //动画立即停止  停在当前位置 
            Log.e(TAG, "onAnimationCancel");  
        }  
    });  
    anim.start();

如果不想实现全部发方法，可以使用

  valueAnimator.addListener(new AnimatorListenerAdapter() {
    @Override
    public void onAnimationEnd(Animator animation) {
        super.onAnimationEnd(animation);
    }
});

这样就可以选择实现要监听的方法。

AnimatorSet

这个在前面有提过，是用来执行动画合集的类，简单使用方法：

ObjectAnimator animator = ObjectAnimator.ofFloat(mTestImage, "scaleX", 1.0f, 0f);
   ObjectAnimator animator1 = ObjectAnimator.ofFloat(mTestImage, "scaleY", 1.0f, 0f);
   AnimatorSet animatorSet = new AnimatorSet();
   animatorSet.setDuration(1000);
   //animatorSet.playSequentially();顺序执行
   animatorSet.playTogether(animator, animator1);
   animatorSet.start();

还有一种控制执行顺序的方法：

AnimatorSet animSet = new AnimatorSet();  
    animSet.play(anim1).with(anim2);  
    animSet.play(anim2).with(anim3);  
    animSet.play(anim4).after(anim3);  
    animSet.setDuration(1000);  
    animSet.start();
