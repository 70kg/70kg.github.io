使用Hexo建站的那些坑

发表于 2015-06-14   |   分类于 Hexo   |  

  这段时间学习使用Hexo搭建这个博客，从一个什么都不懂的小白倒腾了半天弄出了这个博客，过程中遇到了很多坑，写下来不然过段时间就忘了，毕竟好记性不如烂笔头。
  首先先感谢一下网上的那些教程，先贴一下他们的链接吧，没有他们就没有今天我的这个网站（好感人。。55）。
http://zipperary.com/categories/hexo/
http://cnfeat.com/2014/05/10/2014-05-11-how-to-build-a-blog
http://ibruce.info/2013/11/22/hexo-your-blog/
  我的环境的Mac，win的环境还没配置过，下面说一下我遇到的那些坑吧

安装hexo出错

这步出现在使用nmp安装hexo的时候。具体的报错也忘记了，也懒得再去重现一遍。这时你可以尝试使用sudo npm install --unsafe-perm --verbose -g hexo在sudo下进行安装。

添加npm依赖包

这步是在使用npm添加依赖包出现错误，你也可以尝试在sudo下进行此命令，sudo npm install

ERROR Deployer not found: git

出现这个错误具体在哪我也忘了，这也说明了记录下来的重要性。当时只是简单的mark了一下错误的解决办法。如果你遇到了这个错误，尝试使用这个命令sudo npm install hexo-deployer-git --save

  好了，这就是我遇到的那些坑，我还在学习的路上，以后肯定还是会遇到更多的坑，也会努力从不同的坑中爬出来。我也是使用OSX系统没多久，以前在win下也没怎么使用过命令行，现在发现命令行还是很方便的，开始喜欢上了。

  还有个问题，托管在GitHub上之后，每次提交就会删掉CNAME文件，我还得自己手动去添加，很是麻烦，放在source下面又会生成html文件，如果有人碰巧看到并且知道怎么解决麻烦告知我一下，感谢！

8.22更新

今天又翻到这篇，现在已经把博客同时托管到了gitcafe上，国内访问速度刚刚的，而且把图床也交给了七牛。至于CNAM域名的问题，因为现在这个站点也就我自己用，一个月能有两三个独立IP访问就不错了，我也懒得再去github上去搞。至于为什么还要每次push到github上，是因为我在刷签到次数啊😂😂。现在也增加了百度的统计功能，但是 基本上 所有的PV都是我自己😂。也是惨的不行不行的
